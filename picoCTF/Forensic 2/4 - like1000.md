## like1000
## Descripcion
This [.tar file](https://challenge-files.picoctf.net/c_fickle_tempest/4e504d8229a455fa2c35c2c29c109d92388299b76a4fb323438c2b621a82e329/1000.tar) got tarred a lot.
## Solucion

- Verificamos que tipo de archivo es :

`file 1000.tar`

- Vemos el contenido sin descomprimir, vemos que es un multiple tar con varias capas, presumiblemente 1000

`tar -tf 1000.tar`

- descomprimimos y vamos borrando los archivos anteriores

`for i in {1000..1}; do tar -xf $i.tar && rm $i.tar ; done`

- abrimos la bandera

`open flag.png`
PICOCTF{l0t5_0f_TAR5}
## Notas
- **`file <archivo>`**: Es fundamental porque las extensiones pueden mentir. Este comando lee la "firma" o _magic bytes_ del archivo para decirte qué es realmente (un tar, un zip, una imagen, etc.).
    
- **`tar -tf`**: La `t` significa _list_. Te permite ver el "índice" del archivo sin extraerlo. Es útil para predecir el nombre del siguiente archivo en la secuencia.
    
- **El bucle `for`**: En este caso, se usa un **bucle de rango decreciente** `{1000..1}`. Es eficiente porque:
    
    - `tar -xf`: Extrae el contenido.
        
    - `&&`: Asegura que el archivo anterior solo se borre (`rm`) si la extracción fue exitosa.
## Referencias
#### Cuando no sabes el número exacto de capas

Si no sabes que son 1000, un bucle `for` fallará. En ese caso, se usa un bucle `while` que se ejecute mientras el archivo de salida sea un comprimido:

Bash

```
# Ejemplo lógico: Mientras el comando 'file' diga que es un archivo comprimido
while [ -f *.tar ]; do
    FILE=$(ls *.tar)
    tar -xf "$FILE" && rm "$FILE"
done
```

#### Diferentes tipos de compresión

A veces las capas alternan entre `.zip`, `.tar`, `.gz` y `.bz2`.

- **Referencia de herramienta:** `7z` (p7zip) es muy potente porque puede extraer casi cualquier formato con el mismo comando (`7z x archivo`).
    
- **Scripting:** Puedes usar un script que detecte la extensión y ejecute el comando correspondiente (`unzip`, `tar -xf`, `gunzip`, etc.).
